# software_development_life_cycle

本目錄提供軟體開發生命週期（SDLC）的可召喚專家集合，聚焦需求、架構、實作、驗證與運維等核心技術場景。

## 目的與定位

- 主軸：技術決策與工程交付。
- 範圍：從需求釐清到交付運維的全技術鏈路。

## 主要場景分組

- 定義與需求：`business_requirement_analysis`、`complexity_tradeoff_analysis`
- 架構與契約：`frontend_system_design`、`backend_system_design`、`api_contract_documentation`、`data_migration_evolution`
- 實作與交付：`frontend_development`、`backend_development`、`security_compliance`
- 驗證與品質：`testing_strategy`、`system_integration_validation`、`performance_scalability`
- 運維與穩定：`observability_operations`

## 專家清單（SDLC）

- business_requirement_analysis: 需求邊界、驗收條件、業務規則
- complexity_tradeoff_analysis: 複雜度與技術取捨
- frontend_system_design: 前端資訊架構與互動模型
- backend_system_design: 後端領域模型與服務邊界
- api_contract_documentation: API 契約、版本相容、Breaking Changes
- data_migration_evolution: Schema 演進與資料遷移策略
- frontend_development: 前端實作、除錯與優化
- backend_development: 後端服務實作與資料存取邏輯
- testing_strategy: 測試金字塔、覆蓋率與品質閘門
- system_integration_validation: 跨系統整合驗證與韌性測試
- performance_scalability: 效能瓶頸、容量擴展與壓測
- security_compliance: 威脅建模、資安與合規檢核
- observability_operations: SLI/SLO、告警調優與可觀測性
