# enterprise_delivery_life_cycle

本套件用於補齊企業網站專案生命週期中的非純技術決策與溝通場景，並與 SDLC 技術專家互補。

## 目的與定位

- 主軸：溝通、協商、交接、關係經營等 Soft-Power 場景。
- 範圍：從售前到結案續約的跨角色決策、治理與協作。

## 主要場景分組

- 售前與決策敘事：`presales_persuasion_skills`、`presentation_storyboarding`
- 治理與協作：`program_project_governance`、`stakeholder_conflict_resolution`、`collaborative_communication`
- 變更與控制：`change_request_management`、`risk_issue_dependency_governance`、`report_writing`
- 交付控制與品質：`quality_delivery_governance`、`specification_traceability_governance`、`resource_capacity_governance`
- 驗收與交付閉環：`persuasion_negotiation`、`transition_operations`
- 採納與關係經營：`adoption_change_enablement`、`relationship_management`
- 商務與風險控制：`commercial_delivery_governance`

## 邊界原則

- `program_project_governance` 處理治理節奏與 Gate 決策；`stakeholder_conflict_resolution` 處理框架內的仲裁。
- `risk_issue_dependency_governance` 處理 RAID 原始數據；`report_writing` 將數據轉化為管理摘要。
- `quality_delivery_governance` 負責品質門檻與 DoD；`specification_traceability_governance` 負責需求到規格的追溯治理。
- `resource_capacity_governance` 負責團隊容量規劃與預估準確度。
- `transition_operations` 整合上線與交接兩階段；不取代 `adoption_change_enablement` 的行為轉換推動。

## 結構原則

- 每個 `REASONING.md` 應盡量使用一致骨架：`核心價值觀`、`互動準則`、`核心工具箱`、`與其他專家的邊界`、`適用原則`。
- `與其他專家的邊界` 使用正向定義說明專家責任邊界，取代過往負向「不適用場景」。

## 專家清單（Enterprise Soft-Power）

- presales_persuasion_skills: 售前價值主張與提案勝率
- presentation_storyboarding: 高階決策簡報故事線與逐頁結構
- specification_traceability_governance: 需求到規格的可追溯性與 SD/SA 品質治理
- program_project_governance: stage gate、治理節奏與決策論壇設計
- stakeholder_conflict_resolution: 跨部門衝突與優先級仲裁
- change_request_management: baseline 後變更治理與衝擊控制
- collaborative_communication: 跨團隊協作節奏與交接品質
- risk_issue_dependency_governance: RAID 治理、升級路徑與追蹤閉環
- persuasion_negotiation: UAT 驗收談判與承諾邊界
- report_writing: 管理層決策摘要與證據鏈
- transition_operations: 上線切換與營運交接閉環
- adoption_change_enablement: rollout 溝通、使用者採納、訓練與內部推廣 (Stage 分流)
- relationship_management: 客戶成功、續約風險與關係經營
- commercial_delivery_governance: SOW 邊界、商務條款、罰則風險與分包商管理
- quality_delivery_governance: 品質閘口 (Quality Gate)、DoD 與驗收標準對齊
- resource_capacity_governance: 人天預估、團隊容量規劃與資源爭用治理

**總計**: 16 個專家（v2.2）
